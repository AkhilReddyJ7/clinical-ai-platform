# 0003: Alembic migrations over implicit `create_all`

- **Status:** Accepted (supersedes an initial MVP shortcut)
- **Date:** 2026-07-03

## Context

Persistence first shipped using `Base.metadata.create_all()` in the FastAPI
lifespan hook — the fastest path to a working MVP. This broke in practice:
editing a file under the docker-compose bind-mounted repo triggered
uvicorn's `--reload`, which reran the lifespan hook mid-migration and threw
a `DuplicateTableError` against the live Postgres instance. That was direct
evidence the implicit-reflection approach was actively wrong, not just
theoretically inferior to a real migration tool.

## Decision

Adopt Alembic as the sole owner of database schema. `alembic/env.py` reads
`DATABASE_URL` from the application's own `Settings` object (no parallel
configuration to keep in sync) and reuses the existing async SQLAlchemy
engine — no second database driver was added. `docker-compose.yml` runs
`alembic upgrade head` before `uvicorn` starts; the FastAPI lifespan hook no
longer touches schema at all.

`DocumentStatus` is stored as `VARCHAR` (`native_enum=False`) rather than a
native Postgres `ENUM` type, specifically so that adding a new status value
in the future is a plain column-constraint change, not an
`ALTER TYPE ... ADD VALUE` migration.

## Consequences

- Every model change now requires `alembic revision --autogenerate -m "..."`
  plus a manual review of the generated migration — real, permanent
  workflow overhead compared to the implicit approach.
- Schema history is now reviewable and reversible in principle, not just
  reproducible from the current state of the ORM models.
- The initial revision (`6751263bc707`) was verified by applying it to a
  genuinely empty Postgres instance, not just by inspection — see
  [0004](0004-sqlite-for-tests-postgres-for-runtime.md) for why this
  verification did not happen through the automated test suite.
- Known gap: the autogenerated `downgrade()` for the initial revision has
  never actually been executed, only generated. Real risk the first time a
  rollback is genuinely needed.
